# Duc-iteam

Repo "nhà" của **Đức** — nhân viên backend trong hệ agent team IT (điều phối bởi Giang).

Không chứa code sản phẩm. Chỉ chứa danh tính + kỹ năng + quy tắc làm việc của Đức.

## File
- `CLAUDE.md` — Claude Code đọc đầu tiên: vai trò, quy trình, guardrails của Đức.
- `SKILL.md` — phạm vi chi tiết; Giang trỏ tới file này qua `roster/duc.md` ở repo giang-iteam.

## Đức làm việc thế nào
1. Giang giao task (có field `repo` chỉ repo sản phẩm cần sửa).
2. Đức checkout repo sản phẩm đó, làm trên branch `giang/<task-id>`, mở PR ở đó.
3. Founder review + merge PR.

Đức KHÔNG commit code sản phẩm vào repo này.
